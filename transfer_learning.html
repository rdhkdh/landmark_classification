Question: Outline the steps you took to get to your final CNN architecture and your reasoning at each step. Describe why you think the architecture is suitable for the current problem.

Answer:

  • ResNet's deep architecture with residual connections, its proven performance on image classification tasks, and the availability of pretrained models make it an excellent choice for landmark classification tasks. ResNet18 has been used in this particular task.
  • By using a pretrained ResNet model, we can leverage the learned features from a large, diverse dataset like ImageNet, which helps in better generalizing and distinguishing between the 50 different landmarks, even with a limited amount of task-specific training data.
  • When classifying images into 50 notable landmarks, the network needs to distinguish between a wide variety of architectural styles, natural formations, and other unique landmark features. ResNet's ability to learn deep and detailed feature representations makes it highly effective for this task.
  • Why is it preferred over other models? VGGNet, DenseNet etc are also available, but these models have higher computational cost in terms of memory and CPU usage; and suffer from vanishing gradient problem. ResNet mitigates the vanishing gradient problem by introducing skip connections; is readily available pre-trained, on large datasets of natural images; and has a good balance between depth, parameter efficiency and cost.

from src.predictor import Predictor
from src.helpers import compute_mean_and_std

# First let's get the class names from our data loaders
class_names = data_loaders["train"].dataset.classes

# Then let's move the model_transfer to the CPU
# (we don't need GPU for inference)
model_transfer = model_transfer.cpu()
# Let's make sure we use the right weights by loading the
# best weights we have found during training
# NOTE: remember to use map_location='cpu' so the weights
# are loaded on the CPU (and not the GPU)
model_transfer.load_state_dict(
    torch.load("checkpoints/model_transfer.pt", map_location="cpu")
)

# Let's wrap our model using the predictor class
mean, std = compute_mean_and_std()
predictor = Predictor(model_transfer, class_names, mean, std).cpu()

# Export using torch.jit.script
scripted_predictor = torch.jit.script(predictor)
scripted_predictor.save("checkpoints/transfer_exported.pt")
